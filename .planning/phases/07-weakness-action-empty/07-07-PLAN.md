**Note on ROADMAP constraint:** ROADMAP states "최대 2개 병렬 쿼리 (sessions + feedback). snapshotsJson 집계뷰에서 제외". The third query is endorsed by CONTEXT.md and RESEARCH.md for body language panel (WEAK-03). It is NOT included in the aggregation/analytics compute — it feeds a separate `computeBodyLanguage` call. The ROADMAP constraint about "집계뷰에서 제외" is preserved: snapshotsJson is not part of `computeAnalytics`. The query limit constraint is superseded by the phase design decision.

1. Add import for metricSnapshots:
```typescript
import { feedback, metricSnapshots, sessions } from "@/shared/db/schema";
import { computeAnalytics, computeBodyLanguage } from "@/features/analytics";
```

2. Expand the existing `Promise.all` destructuring from `[userSessions, feedbackRows]` to `[userSessions, feedbackRows, snapshotRows]`, and append a third query:
```typescript
// NEW: metricSnapshots query (latest 2 sessions only, for body language panel)
db
  .select({
    sessionId: metricSnapshots.sessionId,
    snapshotsJson: metricSnapshots.snapshotsJson,
  })
  .from(metricSnapshots)
  .innerJoin(sessions, eq(metricSnapshots.sessionId, sessions.id))
  .where(eq(sessions.userId, session.user.id))
  .orderBy(desc(sessions.createdAt))
  .limit(2),
```

3. After the existing `const analytics = computeAnalytics(serialized, feedbackRows);` line, add:
```typescript
const bodyLanguage = computeBodyLanguage(snapshotRows);
```

4. Add `bodyLanguage` prop to the HistoryDashboard JSX:
```typescript
<HistoryDashboard
  sessions={serialized}
  analytics={analytics}
  bodyLanguage={bodyLanguage}
/>
```

**CRITICAL: Do NOT modify the existing sessions or feedback queries. Only ADD the third query and the computeBodyLanguage call.**
</action>
