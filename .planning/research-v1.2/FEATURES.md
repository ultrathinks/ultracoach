# Features Research: v1.2 Coaching Loop

## Table Stakes (Must Have)

### 모범 답안 생성
- **질문별 개선 답안**: 사용자의 실제 답변을 기반으로 "이렇게 말했으면 더 좋았을" 리라이트 제공
- **구체적 개선 포인트**: 단순 리라이트가 아니라 "왜 이 버전이 더 나은지" 설명 포함
- **사용자 맥락 유지**: 사용자의 실제 경험을 기반으로 리라이트 (완전히 새로운 이야기가 아님)
- **STAR 구조 반영**: 부족한 STAR 요소를 채워넣은 버전 제시

### 재연습 드릴
- **단일 질문 집중**: 전체 면접이 아닌 특정 질문 하나에 대해 반복 연습
- **카메라 피드**: 자기 모습을 보면서 연습 (거울 효과)
- **음성 녹음 + 전사**: 실제 말한 내용을 텍스트로 확인
- **즉각 피드백**: 연습 후 바로 LLM 분석 결과 제공

## Differentiators (Nice to Have)

- **Before/After 비교 뷰**: 원래 답변 vs 재연습 답변을 나란히 보여주는 UI
- **반복 횟수 추적**: 같은 질문을 몇 번 연습했는지, 개선 추이
- **핵심 키워드 하이라이트**: 모범 답안의 핵심 포인트가 재연습 답변에 포함되었는지 표시
- **타이머 가이드**: "2분 이내로 답변하세요" 같은 시간 제한 연습

## Anti-Features (Avoid)

- **답변 암기 유도**: 모범 답안을 그대로 외우게 만들면 안 됨. "참고용"임을 명확히
- **과도한 게이미피케이션**: 스코어보드, 뱃지 등은 취업 준비 맥락에서 부적절
- **자동 재연습 강제**: "이 질문 다시 하세요" 팝업은 압박감 유발. 사용자 주도여야 함
- **전체 면접 재연습**: 전체 세션 다시 하기는 v1.2 범위 아님 (기존 새 세션으로 충분)
- **드릴 결과 DB 저장**: ephemeral로 유지. 히스토리 오염 방지

## UX Patterns from Similar Products

### 면접 코칭 앱 패턴
- **Pramp/Interviewing.io**: 피어 리뷰 후 구체적 개선점 제시, 같은 문제 재도전 가능
- **Big Interview**: 답변 녹화 → AI 분석 → 모범 답안 비교 → 재녹화 루프
- **언어 학습 앱 (Duolingo/Elsa)**: 짧은 드릴 → 즉각 피드백 → 반복의 마이크로러닝 패턴

### 핵심 UX 원칙
1. **진입 장벽 최소화**: 결과 화면에서 원클릭으로 드릴 진입
2. **빠른 피드백 루프**: 답변 → 3초 이내 피드백 시작
3. **비교 가능한 형태**: 원래 답변과 개선 답변을 나란히 볼 수 있어야 함
4. **자기 주도**: 어떤 질문을 연습할지 사용자가 선택

## Complexity Assessment

| 기능 | 복잡도 | 근거 |
|------|--------|------|
| suggestedAnswer 생성 | Low | 기존 feedback 프롬프트 확장 + 스키마 1필드 추가 |
| 모범 답안 UI 표시 | Low | report-view.tsx에 collapsible 섹션 추가 |
| 드릴 모드 카메라 | Medium | 기존 interview-screen의 카메라 로직 재사용, Simli 제거 |
| 드릴 VAD + Whisper | Medium | 기존 코드 재사용하되 단일 질문용으로 단순화 |
| 드릴 피드백 API | Low | 새 route지만 기존 패턴 복제 |
| 드릴 결과 UI | Medium | 피드백 표시 + "다시 하기" 루프 UX |

**전체 복잡도**: Medium. 기존 인프라 재사용 비중이 높아 새로 만들 코드량은 적음.
